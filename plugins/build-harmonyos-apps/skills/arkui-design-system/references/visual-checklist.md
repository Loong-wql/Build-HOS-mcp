# ArkUI Visual Checklist

## Review Areas
- Existing tokens: color, type, spacing, radius, elevation, motion.
- State coverage: loading, empty, error, disabled, pressed, selected, focused.
- Accessibility: text scale, contrast, touch target size, semantic labels where needed.
- Responsive behavior: phone, tablet, foldable, and landscape if in scope.
- Localization: Chinese and English expansion, numeric/date formatting, truncation rules.

## Output
Report concrete UI risks and fixes. Do not invent a new visual system when project tokens already exist.
