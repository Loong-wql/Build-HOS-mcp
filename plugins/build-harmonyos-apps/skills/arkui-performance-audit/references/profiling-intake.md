# Profiling Intake

Collect these before diagnosing runtime-only issues:

- Device or emulator model, OS/API version, and build mode.
- Exact app version or commit.
- Exact user flow: start screen, actions, stop condition.
- Whether the symptom reproduces on first launch, warm launch, or after prolonged use.
- Available evidence: DevEco CPU/memory/frame captures, hilog excerpts, screenshots, or app telemetry.

Use the same device, build mode, and flow for before/after comparisons.
