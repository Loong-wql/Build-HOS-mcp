# Performance Report Template

## Flow
Describe the exact screen and interaction measured.

## Evidence
List build mode, device/emulator, logs, screenshots, profiler exports, and whether the evidence is direct measurement or code-review suspicion.

## Findings
Order by expected impact. For each finding include:
- symptom;
- code or evidence reference;
- likely cause;
- suggested fix;
- validation step.

## Caveats
Call out simulator-only cost, missing release build data, low sample count, network variance, or incomplete profiler exports.
