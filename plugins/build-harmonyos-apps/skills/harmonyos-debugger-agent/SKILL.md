# HarmonyOS Debugger Agent

## Overview
Use `harmonyosmcp` to build the current HarmonyOS project, list connected devices, install a HAP, launch a UIAbility, inspect hilog output, capture screenshots, dump UI layout, and inject basic UI input. Prefer MCP tools over ad hoc shell commands when the tool is available.

This is the HarmonyOS counterpart to the iOS debugger workflow, but the parity is not exact: XcodeBuildMCP exposes richer simulator UI automation than stock public HDC workflows normally do. HarmonyOS input automation here is based on device-side `uitest uiInput` and layout dumps.

## Core Workflow

### 1. Discover the project
- Call `harmonyos_find_project` with the repo or feature path.
- Call `harmonyos_inspect_app_metadata` to infer bundle name, modules, and abilities from `app.json5` / `module.json5`.
- Confirm the inferred root contains `oh-package.json5`, `build-profile.json5`, or `hvigorw`/`hvigorw.bat`.
- If multiple HAP outputs exist, pick the one generated by the current build or ask the user which product flavor is intended.

### 2. Build the app
- Call `harmonyos_build` with `task: "assembleHap"` unless the repo documents a different task.
- If the project needs a product, module, or signing mode, pass repo-specific arguments through `extraArgs` instead of guessing.
- Treat a non-zero exit code as a failed build. Read the stderr/stdout before trying to install.

### 3. Select a device or emulator
- Call `harmonyos_list_devices`.
- If more than one target is connected, use the explicit `targetId` returned by HDC.
- If no target is available, ask the user to start a DevEco emulator or connect a HarmonyOS device.

### 4. Install and launch
- Prefer `harmonyos_build_install_launch` when the user asks for the full run flow.
- For narrower work, call `harmonyos_install_hap` with the chosen `.hap` path, then `harmonyos_launch_ability`.
- Use bundle name and ability name from `harmonyos_inspect_app_metadata`, `module.json5`, `app.json5`, or project documentation.
- Default ability name is usually `EntryAbility`, but confirm from source when possible.

### 5. Verify and interact
- Capture logs with `harmonyos_hilog`, filtered by bundle name, ability name, or app log tag.
- Capture a screenshot with `harmonyos_screenshot` or `harmonyos_ui_screenshot` when visual confirmation matters.
- Dump the UI tree with `harmonyos_dump_layout` before relying on coordinates.
- Use `harmonyos_tap`, `harmonyos_swipe`, `harmonyos_press_key`, and `harmonyos_type_text` for basic interaction once the target is known.
- Report the exact build artifact, target id, launched bundle/ability, and evidence paths or log excerpts.

## Tool Summary
- `harmonyos_find_project`: locate project markers and HAP outputs.
- `harmonyos_inspect_app_metadata`: infer bundle name, modules, and abilities.
- `harmonyos_build`: run hvigor tasks.
- `harmonyos_build_install_launch`: run the full build/install/launch flow.
- `harmonyos_list_devices`, `harmonyos_install_hap`, `harmonyos_uninstall`, `harmonyos_launch_ability`, `harmonyos_hilog`, `harmonyos_screenshot`: focused device actions.
- `harmonyos_ui_screenshot`, `harmonyos_dump_layout`: `uitest`-backed visual and UI tree evidence.
- `harmonyos_tap`, `harmonyos_swipe`, `harmonyos_press_key`, `harmonyos_type_text`: `uitest uiInput` / `uinput`-backed basic UI automation.

## Troubleshooting
- If HDC cannot find a target, confirm SDK environment variables and that `hdc list targets` works outside Codex.
- If install fails, check signing, API level compatibility, and whether an older package must be removed first.
- If launch fails, verify bundle name and ability name from project metadata rather than assuming defaults.
- If screenshot capture fails, the device image may not include `snapshot_display`; try `harmonyos_ui_screenshot` because it uses `uitest screenCap`, or fall back to DevEco Studio and state that limitation.
- If input automation fails, confirm the target shell supports `uitest uiInput` with `uitest uiInput help`; Android-style `input` may not exist on HarmonyOS images.
